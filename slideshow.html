Case Overview

What we were asked to do

Trade 13 instruments across a simulated exchange, maximising profit-and-loss over 15-minute rounds, competing against other university teams and exchange bots.

A / B / C

Three equities

A: earnings-driven small-cap. B: semiconductor (no direct news). C: insurance with Fed-rate sensitivity.

ETF

Basket of A + B + C

Swap mechanism: convert 1 ETF ↔ 1A + 1B + 1C for a fee of 5 ticks.

Options

6 European options on B

Calls and puts at strikes 950, 1000, 1050. Single expiry at round end.

Fed

3 prediction contracts

R_HIKE / R_HOLD / R_CUT. Settle at 1000 or 0. Fed decision driven by CPI news and fedspeak headlines.

Scoring formula

Score = (n − 1)²

Where n is your leaderboard position that round. Rank 1 → score 0. Rank 7 → score 36. Rank 42 → score 1681.

The quadratic penalty means one catastrophic round costs more than ten solid rounds gain. Consistency, not peak PnL, wins.

Phase 1 — Early Iterations

The problems we hit immediately

🔴 Positions at ±200

Every asset was hitting the exchange hard limit. Orders being rejected. No inventory control.

A: −200 R_HIKE: +200 R_CUT: −200

Root cause: No fair value → one side of our quotes picked off continuously.

🔴 PnL: negative → positive arc

Every round started deeply negative and slowly recovered. Classic adverse selection pattern.

T+0s: FV = None → use book mid
T+0→22s: smart bots pick off quotes
T+22s: first earnings → FV known
T+22→: recover slowly via sweeps

🔴 Options at zero

Box spread and PCP arb never executed. Options positions stayed flat all round.

Why? 4-leg friction = 12 ticks. Box value = 50 ticks. Need 24% mispricing — never happens.

Buy box cost = 50 + 12 = 62 > 50 ✗

🔴 C generates nothing

Gradient descent for BETA/GAMMA never converged in 15 minutes. ERC barely varied in early minutes → gradients ≈ 0.

🔴 Fed: lottery ticket

max(1, ...) bug in _kelly_size + wrong position cap = ±200 Fed positions. Win big or lose catastrophically.

🔴 C FV corrupted at round start

Stale prior-round C fair value triggered immediate sniping before the new round's model was ready.

Root Cause Analysis

Why strategy.py ranked first

What it got right

✓ A sweep

Unconditional book sweep immediately after every EPS release. No throttling, no cap on pending orders. Captures the full post-earnings dislocation.

✓ Flow tracker

ALPHA = 0.05 — monitors aggressive trade prints to detect smart money direction. Adjusts quotes when bots know something.

✓ Pre-cancel timer

Cancels all A quotes 3 seconds before the next expected earnings. Prevents being picked off by faster bots who've already priced the new EPS.

✓ Simple + fast

No heavy logging, no stale-order cleanup, no PnL attribution. Low latency means more fills in the post-earnings 2-second window.

What it got wrong (hidden bugs)

Bug: Fed position accumulated to ±200

Asset Deep Dive — Stock A

The primary alpha engine

Why A generates the most PnL

EPS fires at deterministic times: second 22 and second 88 of every 90-second day. That's 20 events per round. The new FV is EPS × PE_A × EPS_SCALE_A. Stale book orders are now mispriced. We sweep them.

FV = EPS × 10 × EPS_SCALE_A
EPS_SCALE_A auto-calibrated from mid/(eps×PE)

The sweep mechanism

Post-earnings, all asks below FV − 2 are swept (BUY). All bids above FV + 2 are swept (SELL). Unconditional — we take these regardless of current position, up to the 200-unit limit. Then we re-quote tightly for 2 seconds.

Smart pre-cancellation

EarningsTimer tracks the next expected earnings (66s or 24s away). Exactly 3 seconds before it fires, we cancel all A quotes. Prevents faster bots from picking off our stale quotes the moment new EPS is broadcast.

Auto-calibration of EPS_SCALE_A

# On every earnings event:
implied = mid / (eps × PE_A)
scale_obs.append(implied)

if len(scale_obs) >= 3:
  median = sorted(scale_obs)[n//2]
  if |median - EPS_SCALE_A| > 1.0:
    EPS_SCALE_A = median

If initial scale is wrong, the first sweep goes in the wrong direction. Auto-calibration self-corrects within 3 events using the median (resistant to a single bad mid reading).

Passive MM between events

Between earnings: quote at FV ± spread, with inventory skew (shift quotes away from the side we're heavy on) and flow adjustment from FlowTracker (ALPHA=0.05).

At pos=0: bid=FV−4, ask=FV+4
At pos=+80: bid=FV−16, ask=FV−4 ← sell pressure
At pos=+200: bid=0 (suppressed), ask=FV+4
Asset Deep Dive — Federal Reserve Markets

From lottery ticket to consistent edge

The bug that caused ±200 positions

# OLD — accumulates forever
def _kelly_size(pos, MAX_POS=25):
  target = int(0.5 * kf * MAX_POS)
  return max(1, min(target, MAX_POS - pos))
# At pos=25: max(1, 0) = 1 → places order
# safe_place_order allows up to 200
# → fires every 0.1s → ±200 in minutes

# NEW — stops at limit
return max(0, min(target, MAX_POS - pos))
# At pos=25: max(0, 0) = 0 → no order

The signal sources

CPI

Actual > Forecast: hawkish → raise R_HIKE probability. Actual < Forecast: dovish → raise R_CUT probability.

Fedspeak

Keyword parsing: "restrictive/hawkish/tightening" → hawkish. "easing/cooling/dovish/cut" → dovish. "balanced/weighing" → neutral.

Book mid

Seeds our initial probability estimates from the market on the first 3 readings. Market consensus as prior.

Half-Kelly sizing

edge = our_p − market_p
kelly_fraction = edge / (1 − market_p)
size = floor(0.5 × kf × MAX_POS)
Capped at MAX_POS=25 per contract

Trade only when divergence > 4%. Size proportional to conviction. Half-Kelly (not full) protects against model error.

Settlement trade (new)

Fires once per round in the T−90s to T−20s window. Only triggers when:

our_p ≥ 0.78 EV positive: ask < 1000 × our_p Up to 35 units

At round end, all CPI prints and fedspeak have accumulated. Our probability estimate is most accurate → largest Kelly bet justified.

15-second cadence

Fed trade() runs every 15s in the main loop (was every 0.1s, creating 9,000 order attempts/round). But fires immediately on CPI and fedspeak news to preserve alpha.

Asset Deep Dive — Stock C

Replacing gradient descent with direct calibration

Why gradient descent failed

The old model fit BETA and GAMMA via 150-iteration gradient descent on (ERC, C_mid, EPS) observations.

The problem: ERC only varies after CPI prints. In the first 3-4 minutes, ERC ≈ constant → gradient of loss w.r.t BETA ≈ 0 → parameters never update → C_FV is garbage → bot accumulates large wrong-direction C position before model corrects.

dC/d(BETA) ∝ ERC ≈ 0 before first CPI
→ 150 iterations change nothing useful

The new 1-parameter model

C_FV = anchor_mid + sensitivity × (ERC − anchor_ERC)

The full formula is linear in ERC for small deviations. We measure the coefficient directly from the first significant CPI event.

# First CPI with |ΔERC| > 2.0:
sensitivity = ΔC_mid / ΔERC
# One observation. Done.
C_FV = anchor + sensitivity × ΔERC

Three-phase lifecycle

Phase 1 — Before first C earnings

No FV. Wide passive MM (±12 ticks, 3 units). No directional trades. Zero risk exposure.

Phase 2 — After first C earnings

PE_C_scale calibrated (=mid/eps). Still no ERC anchor. Wide passive MM continues.

Phase 3 — After first significant CPI

Sensitivity calibrated. FV active. Tight MM ±5 ticks/8 units. Directional C trades on every subsequent CPI (±10 units).

CPI directional trades

On each CPI after calibration, if |expected ΔC| > 4 ticks, we trade before C reprices:

Hawkish CPI: ERC rises → C falls → SELL C at bid
Dovish CPI: ERC falls → C rises → BUY C at ask

Expected: 5-7 CPI events post-calibration × ~8 ticks edge × 10 units = +5k–12k additional PnL/round.

Asset Deep Dive — ETF & Options

Arbitrage, NAV pricing, and the friction problem

ETF strategy

NAV = A_fv + B_implied + C_fv
B_implied from PCP across all 3 strikes
One-sided

ETF > NAV + 11: just SELL ETF. ETF < NAV − 11: just BUY ETF. Lower risk than full-swap — trades the mispriced leg only. Rate-limited: 1 per 3s.

Full swap

Only on very large edges (>17 from NAV). Buy A+B+C simultaneously, swap toETF, sell ETF. 10-second abort timeout.

Case packet hint: "ETF is more likely the mispriced leg." So we trade the ETF against the basket, not the basket against the ETF.

Why options arb never triggers

Box buy friction = 4 × 3 ticks = 12
Box theoretical value = K2 − K1 = 50
Buy edge = 50 − (50 + 12) = −12 ✗

The exchange's option bots maintain PCP within 1-2 ticks. Getting a 12-tick mispricing that persists long enough to execute 4 legs simultaneously is structurally impossible in this market.

What options actually give us

The PCP formula S = C_mid − P_mid + K provides an implied B price from all three strikes. Weighted average (tighter spreads = higher weight) gives us a clean real-time B fair value — used for ETF NAV calculation.

# PCP → implied B
S_implied = C_mid − P_mid + K
weight = 1.0 / (C_spread + P_spread)
B_implied = Σ(S × w) / Σ(w)

We keep checking for arb opportunities (MIN_EDGE=2) but do not expect PnL from options in normal market conditions.

Event Handling — Critical Path

What happens on a CPI print

Step 1

strat_c.pre_cpi_snapshot()

Captures current ERC and C book mid before Fed processes the CPI. This is the pre-move baseline for calibration and for measuring expected ΔC.

Step 2

strat_fed.on_cpi(forecast, actual)

Updates Fed probability estimates. Surprise > 0 → raise R_HIKE probability, lower R_CUT. Surprise < 0 → reverse. ERC changes as a result.

Step 3

strat_c.recalc_fv()

Recomputes C fair value with the new ERC. No-op until C is calibrated. Once calibrated: C_FV = anchor + sensitivity × ΔERC.

Step 4

await strat_c.on_cpi()

If calibrated: take directional C trade (sell if hawkish, buy if dovish) before C book reprices. If first significant CPI: calibrate sensitivity from observed ΔC/ΔERC.

Step 5

await strat_fed.trade()

Immediately places Kelly-sized Fed orders based on updated probabilities. Bypasses the 15s rate limit because this is news-triggered. Cancels stale Fed orders first, then places fresh ones.

Robustness & Consistency

Why this bot stays consistent as rounds get harder

Model-quality alpha

A's EPS model, C's ERC calibration, and Fed's CPI signals are all information-based. They don't rely on being faster than other bots. In harder rounds with faster opponents, we still have better fair values.

Speed-dependent alpha (sweep) degrades. Model-quality alpha persists.

No catastrophic rounds

The Fed bug was the primary source of -90k to -150k rounds. With MAX_POS=25 and the kelly_size fix, the worst possible Fed outcome is: 25 units × 1000 ticks = -25,000. Manageable.

Position management

Three layers of protection: (1) per-strategy soft limits with inventory skew, (2) safe_place_order hard limits, (3) auto-flatten at T-20s. No position can run away indefinitely.

A: up to ±200 C: ±50 Fed: ±25 (±35 terminal) ETF: ±40

Calibration persistence

EPS_SCALE_A and PE_C_scale persist across rounds. The bot gets smarter over the competition. By round 3, scale observations from rounds 1-2 mean EPS_SCALE_A is rock-solid before the first earnings even fires.

Manual overrides

On competition day, if auto-calibration seems slow, type scale 100 or sens -2.5 to manually set the model parameters from observing the market. Meta market symbols registered live via meta SYM1 SYM2.
